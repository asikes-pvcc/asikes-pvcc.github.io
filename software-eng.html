Chapter 7: Software Engineering

Software Engineering Term Definition
Software Engineering The branch of computer science that seeks principles to guide the development of large, complex software systems
Integrated Development Environment (IDE) A system that combines tools for software development (such as editors, compilers, debugging tool, etc) into a single, integrated package
Modular Implementation An implentation philosophy that maximizes independence among modules and minimizes linages
Unified Modeling Language (UML) Tools developed with the object oriented paradigm in mind>
Use Case Interactions between a system and its users
Use Case Diagram A pictoral representation of the proposed system from a user's point of view
Design Pattern A predeveloped model for solving a recurring problem in software design
Software Quality Assurance (SQA) A group charged with overseeing and enforcing quality control systems
Association for Computing Machinery(ACM) See http://www.acm.org

Software Life Cycle: Development Phase
Requirements Analysis The process of specifying what services a system will provide, how the outside world will interact with the system, and any conditions associated with development
Design Creation of a construction plan of the proposed system and a detailed description of the software's system structure
Implementation The process of writing programs, creation of data files, and development of databases
Testing The process of debugging the created programs and confirming that the final software product is compatible with specifications
Data Relationships
One-to-one Associations in which one instance of one class is associated with one instance of another class
One-to-many Associations in which one instance of one class is associated with many instances of another class
Many-to-many Associations in which many instances of one class are associated with many instances of another class
Software Testing
Glass Box Testing A form of testing in which the software tester is aware of the interior structure of the software and uses this knowledge when designing the test
Black Box Testing A form of testing which does not rely on knowledge of the software's interior composition
Beta Testing Testing in which a preliminary version of the software is given to a segment of the intended audience with the goal of learning how the software performs in real-life situations before the final version of the product is solidified and released to the market.
Alpha Testing Similar to beta testing, but occurs at the developer's site
Documentation
User Documentation Explains the features of software and how to use them. Designed to be read by the user of the software
System Documentation Describes the software's internal composition so that the software can be maintained later in its lifecycle
Technical Documentation Describes how a software system should be installed and serviced
Intellectual Property Law Allows the developer of a product to exercise ownership rights over the product
Software License A legal document in which the rights of the developer are formally expressed
Software Disclaimer A statement disclaiming responsibility for damages incurred as a result of software use (note: Courts rarely recognize disclaimers)
Best Defenses Against Software Liability Claims Application of sound software engineering principles and care in development as well as production and maintenance of records to validate these efforts
