Chapter 4: Networking and the Internet

Network Classifications Definitions
LAN Local area network
MAN Metropolitan area network
WAN Wide area network
Network models
Bus network A network topology in which machines are connected to a common communication line (the bus)
Star network A network topology in which one machine out of several serves as the network’s central focal point
Protocols
CSMA/CD Ethernet A protocol which dictates that each message in a bus network be broadcast to all machines on the bus
CSMA/CA Wifi A similar protocol to CSMA/CD ethernet, but with policies to avoid transmission collisions
Combining networks
Repeater A device which passes signals back and forth between two buses, but does not consider the meaning of the signals.
Bridge Connects two buses, but (unlike a repeater) only forwards a message across a connection when the message is directed at a computer on the other side.
Switch Like a bridge, but containing multiple connections, which produces a network.
Router Special purpose computers which provide links between networks and forward messages, but allow each network to maintain its own characteristics
Gateway The point at which one network is linked to an internet
Client/server model
P2P: peer-to-peer A network application model, usually involving temporarily executed processes, in which processors provide and receive service from one another
Distributed system A modern software system consisting of software units which execute as processes on different computers.
Cloud computing Large pools of shared computers on a network which can be accessed and used by clients as needed
Internet Addressing
Tier 1 The top of the ISP network hierarchy, consisting of very high speed, high capacity, international WANs
Tier 2 Mid tier of the ISP network hierarchy, composed of moderately high speed, regional WANs
Tier 3 Independent, generally local ISP which provides access to individual machines
IP address
ICANN A nonprofit corporation established to coordinate the Internet’s operation (see https://www.icann.org)
DNS Internet-wide directory system comprised of name servers
TLD: Top-level domain The suffix following the period in the domain name (e.g. com, edu, co.uk etc)
Country-code TLDs Two letter codes for domains in specific countries (see https://www.worldstandards.eu/other/tlds/ )
Internet protocols
FTP A protocol designed to list and transfer files across a network
HTTP Protocol used to connect users to other servers via the Internet or LAN
SMTP A way two computers on a network may interact when transmitting an email message
World Wide Web
URL A unique address containing the information needed to contact a server and request a document
Client-side activities Activities and inputs provided by a client (e.g. a browser)
Server-side activities Activities and inputs provided by a server
Internet four-layer software
Application layer Consists of software units such as clients and servers that use Internet communication to carry out tasks
Transport layer Accepts messages from the application layer and divides them into smaller packets to ensure proper formatting
Network layer Determines where a packet from the transport layer should be sent using a forwarding table
Link layer Transfers the packet to its destination
Internet is
Packet-switching network Digital data is sent by breaking data into packets, which contain both the data being transmitted and the control information for routing
Fault tolerant Because routing is redundant, the Internet is more reliable and can scale to more people and devices
Redundant Even if tier 1 routers fail, all or most of the network remains connected since there is more than one way to route data
Cybersecurity
Malware Malicious software
Virus Software that infects a computer by inserting itself into programs that already reside in a machine
Worm Autonomous program that transfers and replicates through a network
Trojan horse A program which enters a system disguised as a desirable program
Spyware Software which collects information about activities on the device on which it resides and transports that information back to user
Phishing Technique of obtaining information by posing as a legitimate requester and asking for it
DDoS attack The process of overloading many computers with messages
Protection
Firewall A program that filters traffic passing through a network
Spam filter Firewalls designed to block unwanted email
Proxy server A software unit that acts as intermediary between client and server
Antivirus software Software used to detect and remove viruses and other infections
Cryptography The study of sending and receiving messages securely
Public key encryption Uses two keys to secure a message even though the encryption and decryption algorithms are known
